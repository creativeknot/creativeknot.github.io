Thoughts on Coolify

Posted on May 4, 2024 by Arnold Chand

Introduction

So far I’ve been using Coolify for some of my projects and I have been enjoying the process.

I got to hear about it from a couple of devs over on discord and was quite curious to try it myself. But there really wasn’t any reason for me to create a VPS1 machine to try it out, considering the machine requirements. If I wanted to try it out, then it would have to be for my freelance projects.

Luckily, an opportunity did arrive and a client of mine reached out to mention about a billing provider that they wanted to use with their SaaS2 application. This billing provider software was called Lago, a really cool self-hosting provider where I could create my plans, attach some billable metric to these plans and then finally add a customer to a plan. The great thing about this was that I can all these using API3 calls, in addition to using their UI interface.

Deploy Coolify

Before I could setup Coolify, I had to first get a VPS machine. I went with a Linode cloud machine, obtained a 4 CPU/8 GB RAM machine. It was quite the cost at $50 USD, but this is beneficial for me, when I want to start building more projects I would want to host them in the same machine.

note

Please make sure you know what you are running through a curl command that will execute external shell commands. It’s best to inspect the file and make sure it’s not doing any malicious.

Finally, I went thru the installation instructions in Coolify which had me go through the curl command setup. I also went onto Linode and blocked out all the ports except for 22, 80 and 443 - the standard procedure.

Setup Lago

So Coolify has some preset applications that I can use to quickly deploy something. While the list isn’t that vibrant, it’s still nice to see that there are some presets I can use to test some of those apps.

However, Lago wasn’t one of them. But that’s fine, because Coolify can also take a docker image or a docker compose file. In the case for Lago, I made use of their docker-compose.yml file. With that, I was just able to copy and paste the contents into Coolify and got the service working quite quickly.

It was a fairly easy task to get the docker compose to work. Here are some problems that I had after deployment of Lago via a docker compose:

  • Public access was still restricted - Turns out I have to add a FRONT_PORT and an API_PORT in the environment variables for Coolify to assign it the ports it needs to make it public. So if I don’t add a domain access to these services then I could still use my public IP provided by Linode to access them.
  • Being able to access via a sub-domain - I fixed this by going to the docker service settings in the Service Stack section and then adding the domain name manually. I had to do this for both the API image and the Front image.
  • The Frontend image of Lago couldn’t pick up the endpoint of the Lago API image - I fixed that under the environment variables. Adjust the LAGO_API_URL and LAGO_FRONT_URL to the domain I assigned in the service stack section.

Conclusion

And that was it! I was able to get a Coolify instance running on a Linode VPS machine so I could host a Lago service for my client :)

There isn’t much to say besides me just getting these things setup for my client, but all in all I really like the process and I would like to try some of the preset applications that come with Coolify or try some of my own projects as well.
